# assignment4_intro_to_databases_cs301
Ми створили систему управління лікарнею - HospitalDB, яка налічує 10 таблиць (5 з них було створені на 1 практичному,
а інші 6 додані нові).
Hospitals - основна таблиця, де зібрані усі лікарні з рейтингом, ID та кількістю місць. 
Diagnosis - таблиця з усіма діагнозами та описами до них
Locations - таблиця з ID лікарні та адресами. Використаний зв'язок 1:1
Finance - основна таблиця для керування фінансами лікарень, вказаний дохід та середній дохід розрахований на 1 особу. Використаний зв'язок 1:1
Patients - таблиця, де зібрані усі пацієнти з контактами
Doctors - таблиця з усіма лікарями та їх спеціалізаціями
Appointments - таблиця з усіма записами пацієнти-лікарі
Procedure - таблиця з усіма процедурами, ціною до них та кабінетами
Pharmacy - аптека прив'язана до лікарні, де міститься усі препарати, ціна та кількість. Використаний зв'язок 1:M
Admin - головна таблиця для адмінів. Використаний зв'язок 1:M
Appoitment_Medicines-таблиця для призначення ліків на прийомі у лікаря. Використаний зв'язок M:M

Також окрім зв'язків ще використані constraints - check для створення обмежень по діапазону, not null щоб точно колонка була не порожня, unique для перевірки унікальних.
Procedure для розрахунку доходу лікарні та середнього доходу шляхом загального поділити на кількість пацєнтів
Оптимізована таблиці завдяки індексами на appointmentID, doctorID, hospitalID, щоб далі було швидко шукати дані при виконанні join

1NF - застосовано, адже усі значення зберігаються в одній комірці і обов'язково є первинний ключ в кожній таблиці
2NF - у таблиці Appointment_Medicines quantity залежить відразу від primary key i foreign key
3NF - в усіх таблицях неключові дані не залежать від інших неключових. HospitalID залежить від лікарні, а не рейтингу чи адреси
